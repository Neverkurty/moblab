# Мобильные лабораторные работы

Репозиторий для лабораторных работ по мобильной разработке (Flutter).

## Структура репозитория

Каждая лабораторная работа находится в отдельной ветке Git:

- **main** (или **lab1**) - Лабораторная работа 1: Калькулятор
- **lab2** - Лабораторная работа 2
- **lab3** - Лабораторная работа 3
- и т.д.

## Переключение между лабораторными работами

### Просмотр всех веток

```bash
# Посмотреть локальные ветки
git branch

# Посмотреть все ветки (включая удаленные)
git branch -a
```

### Переключение на лабораторную работу

```bash
# Переключиться на основную ветку (лабораторная работа 1)
git checkout main

# Переключиться на лабораторную работу 2
git checkout lab2

# Переключиться на лабораторную работу 3
git checkout lab3
```

**Важно:** При переключении веток все файлы в рабочей директории автоматически изменятся на версию из выбранной ветки.

### Создание новой лабораторной работы

```bash
# 1. Убедитесь, что вы на ветке main
git checkout main

# 2. Создайте и переключитесь на новую ветку
git checkout -b lab2

# 3. Внесите изменения в проект (создайте новый проект или измените существующий)
# Например, если нужно создать новый Flutter проект:
# flutter create .

# 4. Добавьте изменения в Git
git add .

# 5. Создайте коммит
git commit -m "Лабораторная работа 2: описание"

# 6. Отправьте ветку на GitHub
git push -u origin lab2
```

### Работа с удаленными ветками

```bash
# Получить все ветки с GitHub
git fetch origin

# Переключиться на удаленную ветку (если она еще не локально)
git checkout -b lab2 origin/lab2

# Отправить изменения в удаленную ветку
git push origin lab2
```

## Текущие лабораторные работы

- **main** - Калькулятор с поддержкой темной/светлой темы

## Полезные команды

```bash
# Посмотреть текущую ветку
git branch --show-current

# Посмотреть историю коммитов текущей ветки
git log --oneline

# Сравнить две ветки
git diff main..lab2

# Удалить локальную ветку (после переключения на другую)
git branch -d lab2

# Удалить удаленную ветку
git push origin --delete lab2
```

## Работа с проектом

После переключения на нужную ветку, работайте с проектом как обычно:

```bash
# Запустить проект
flutter run

# Собрать проект
flutter build

# Установить зависимости
flutter pub get
```
