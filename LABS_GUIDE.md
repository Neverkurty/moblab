# Руководство по работе с лабораторными работами

## Текущая структура

Сейчас ваш проект находится в корне репозитория. Это **Лабораторная работа 1 (Калькулятор)**.

## Способы организации лабораторных работ

### Способ 1: Отдельные папки (РЕКОМЕНДУЕТСЯ)

Каждая лабораторная работа в отдельной папке. Структура будет такой:

```
moblab/
├── lab1_calculator/     # Текущий проект (калькулятор)
│   ├── lib/
│   ├── android/
│   ├── ios/
│   └── ...
├── lab2_название/       # Лабораторная работа 2
├── lab3_название/       # Лабораторная работа 3
└── README.md
```

**Как переключаться:**
```bash
# Работать с лабораторной работой 1
cd lab1_calculator
flutter run

# Работать с лабораторной работой 2
cd lab2_название
flutter run
```

**Добавить новую лабораторную работу:**
```bash
# 1. Создать новую папку
mkdir lab2_название
cd lab2_название

# 2. Создать новый Flutter проект
flutter create .

# 3. Или скопировать существующий проект
# (скопировать все файлы из другого проекта)

# 4. Добавить в Git
git add lab2_название/
git commit -m "Добавлена лабораторная работа 2"
git push
```

### Способ 2: Ветки Git

Каждая лабораторная работа в отдельной ветке Git.

**Создать новую лабораторную работу:**
```bash
# Создать и переключиться на новую ветку
git checkout -b lab2

# Работать над проектом (изменять файлы)
# ...

# Закоммитить изменения
git add .
git commit -m "Лабораторная работа 2"
git push -u origin lab2
```

**Переключиться между лабораторными работами:**
```bash
# Посмотреть все ветки
git branch -a

# Переключиться на лабораторную работу 1
git checkout main

# Переключиться на лабораторную работу 2
git checkout lab2

# Переключиться на лабораторную работу 3
git checkout lab3
```

**Важно:** При переключении веток все файлы в рабочей директории изменятся на версию из выбранной ветки.

## Рекомендации

- **Для Flutter проектов** лучше использовать **Способ 1 (папки)**, так как:
  - Каждый проект независим
  - Легче работать с несколькими проектами одновременно
  - Не нужно переключать ветки при работе
  - Можно запускать разные проекты в разных терминалах

- **Ветки Git** лучше использовать, если:
  - Это разные версии одного проекта
  - Нужна история изменений для каждой лабораторной работы
  - Хотите легко сравнивать версии

## Текущая ситуация

Ваш текущий проект (калькулятор) находится в корне репозитория. Вы можете:

1. **Оставить как есть** - считать корень репозитория лабораторной работой 1
2. **Переместить в папку** - создать `lab1_calculator/` и переместить туда все файлы

Если хотите переместить текущий проект в папку `lab1_calculator`, выполните:

```bash
# В корне репозитория
mkdir lab1_calculator
# Переместить все файлы кроме .git в lab1_calculator/
# (лучше сделать это через файловый менеджер или команды PowerShell)
```
